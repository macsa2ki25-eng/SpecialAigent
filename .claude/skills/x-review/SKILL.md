# 投稿レビュースキル (X + ブログ)

## 目的

`data/posts/pending/` にあるX投稿/ブログ記事を、ユーザーの判断で 承認 / 却下 / 編集 に仕分ける。

## 守ってほしいこと

- 基本は `python -m src.cli review` を起動する
- 対話的プロンプト (y/n/e/s/q) が出るので、ユーザーの選択をそのまま受け流す
- レビュー完了後 `python -m src.cli status` で承認済み件数を表示
- 投稿内容に以下が含まれていたら、承認する前に必ずユーザーに **警告** する:
  - 政治・宗教・特定人物批判
  - 医学的断定 (「治る」「絶対効く」等の薬機法違反)
  - X投稿で 140文字超過 (`over_limit=true`)
  - X本文に URL や外部リンクが入っている
  - ステマ規制違反: 「PR」「広告」表記がない、なのに明らかな商品宣伝
- 自然言語で「これは却下」「これを○○に修正」と指示があれば、対話プロンプトの代わりに JSON を直接編集してよい:
  - 却下 → `data/posts/rejected/` に移動し `status: rejected` を書き込む
  - 承認 → `status: approved` + `approved_at` を書き込む
  - 編集 → `text` または `title` / `body_markdown` を上書き、`char_count` 再計算

## 完了時に必ず伝えること

承認件数・却下件数・残件数を報告し、
「投稿するには `python -m src.cli publish --target all`」と案内する。

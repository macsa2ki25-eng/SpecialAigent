# X 投稿レビュースキル

## 目的

`data/posts/pending/` にある投稿を、ユーザーの判断で 承認 / 却下 / 編集 に仕分ける。

## 守ってほしいこと

- 基本は `python -m src.cli review` を起動する
- 対話的プロンプトが出るので、ユーザーの選択をそのまま受け流す
- レビュー完了後 `python -m src.cli status` で承認済み件数を表示する
- 投稿内容に以下が含まれていたら、承認する前に必ずユーザーに **警告** する:
  - 政治・宗教・特定人物批判
  - 医学的断定 / 投資金額の断定
  - 140文字超過 (`over_limit=true`)
  - URL や外部リンク
- ユーザーが自然言語で「これは却下」「これを○○に修正」と指示したら、対話プロンプトの代わりに JSON を直接編集してもよい
  - 却下 → ファイルを `data/posts/rejected/` に移動し `status: rejected` を書き込む
  - 承認 → `status: approved` + `approved_at` を書き込む
  - 編集 → `text` を上書き + `char_count` 再計算

## 完了時に必ず伝えること

承認件数・却下件数・残件数を報告し、「投稿するには `python -m src.cli publish`」と案内する。
